The current integration surface is centered on Google Sheets export.

## `GET /api/v1/integrations/config`

Return feature availability flags for integrations that are enabled in the running environment.

### Example response

```json
{
  "google_sheets": true
}
```

## `GET /api/v1/integrations/google/auth`

Start the Google OAuth flow.

This endpoint is public, but it is meant to be opened from an authenticated browser flow so the OAuth state cookie can round-trip correctly.

Current behavior:

- Response is an HTTP redirect to Google's consent screen
- The backend sets a short-lived `oauth_state` cookie

## `GET /api/v1/integrations/google/callback`

Finish the Google OAuth flow.

This endpoint requires a signed-in session because the backend needs your user identity before storing Google tokens.

On success, the backend redirects to:

```text
/dashboard/integrations?success=true
```

## `GET /api/v1/integrations/google/status`

Return the current Google integration status for the authenticated user.

### Example responses

Not connected:

```json
{
  "connected": false
}
```

Connected:

```json
{
  "connected": true,
  "email": "name@example.com"
}
```

If the stored connection exists but Google user info cannot be fetched, the backend may still return `connected: true` with an `"Unknown"` email value.

## Related export endpoint

Once Google Sheets is connected, use [Results and exports](/api-reference/results-and-exports) to call:

- `POST /api/v1/jobs/{id}/export/google-sheets`
