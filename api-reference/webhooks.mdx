<Warning>
  This page documents webhook configuration endpoints only. Outbound event payloads and delivery semantics are not published yet, so you should not build against undocumented event contracts.
</Warning>

## Using webhooks with n8n

If you use n8n, create an n8n `Webhook` node and register its **Production URL** in BrezelScraper.

Treat the incoming webhook as a trigger, not as the source of truth for job data. After n8n receives the call, fetch the job or results through the BrezelScraper API.

## `GET /api/v1/webhooks`

List webhook configurations for the authenticated user, including revoked entries.

### Example response

```json
[
  {
    "id": "018efdf7-7fe1-7c42-b4e2-123456789abc",
    "name": "Production webhook",
    "url": "https://example.com/webhooks/brezel",
    "verified_at": "2026-04-10T09:00:00Z",
    "created_at": "2026-04-10T09:00:00Z",
    "updated_at": "2026-04-10T09:00:00Z"
  }
]
```

## `POST /api/v1/webhooks`

Create a new webhook configuration.

### Request body

```json
{
  "name": "Production webhook",
  "url": "https://example.com/webhooks/brezel"
}
```

Rules:

- `name` is required and must be 100 characters or fewer
- `url` is required and must be 2048 characters or fewer
- Maximum active webhook configs per user is 10
- URLs are validated before storage and local or unsafe targets are rejected

### Success response

```json
{
  "id": "018efdf7-7fe1-7c42-b4e2-123456789abc",
  "name": "Production webhook",
  "url": "https://example.com/webhooks/brezel",
  "secret": "f4d5e6c7..."
}
```

The `secret` is returned exactly once.

## `PATCH /api/v1/webhooks/{id}`

Update one webhook configuration.

### Request body

```json
{
  "name": "Renamed webhook",
  "url": "https://example.com/webhooks/new-path"
}
```

Both fields are optional. The request is merged into the existing record.

Current behavior:

- Success response is `204 No Content`
- Invalid UUID values return `400 Bad Request`
- Unknown or revoked IDs return `404 Not Found`

## `DELETE /api/v1/webhooks/{id}`

Revoke one webhook configuration.

Current behavior:

- Success response is `204 No Content`
- Invalid UUID values return `400 Bad Request`
- Unknown or already revoked IDs return `404 Not Found`
