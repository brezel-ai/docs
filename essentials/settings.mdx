## What `docs.json` controls

- Site branding such as `name`, `logo`, `favicon`, and theme colors
- Top-level navigation and sidebar groups
- Global anchors such as the website and pricing links
- Navbar actions such as support and dashboard links

## Project-specific conventions

- Keep customer navigation focused on the top tasks shipping now
- Keep contributor pages hidden unless they are intentionally part of the public site
- Prefer hand-written API pages over stale generated OpenAPI content
- Validate `docs.json` after any change

## Useful fields to review when editing

| Field | Why it matters |
| --- | --- |
| `name` | Controls the global docs title |
| `navigation` | Controls tabs, groups, and page ordering |
| `global.anchors` | Adds website and pricing shortcuts |
| `navbar` | Controls support and dashboard actions |
| `logo` and `favicon` | Keeps branding aligned |

## Safe editing checklist

- Keep JSON valid
- Verify every referenced page path exists
- Remove starter placeholders instead of leaving them hidden
- Re-check links after renaming a page
