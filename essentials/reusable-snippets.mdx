## Good candidates for snippets

- Repeated authentication guidance
- Shared warnings about credits, billing gates, or unpublished webhook payloads
- Repeated support contact language
- Reused definitions for jobs, results, credits, and integrations

## Snippet rules

- Store snippet files under `snippets/`
- Keep snippets short and stable
- Prefer snippets for repeated warnings or definitions, not for large page sections that may drift in meaning

## Basic pattern

```mdx
import MySnippet from '/snippets/my-snippet.mdx';

<MySnippet />
```

Use a snippet when it reduces maintenance. Do not add abstraction if the wording is only used once.
