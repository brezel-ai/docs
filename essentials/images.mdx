## Prefer product proof over decorative media

- Prefer real dashboard screenshots over generic stock images
- Crop tightly around the workflow you are explaining
- Blur or remove private account data, API keys, and email addresses
- Update screenshots when the UI changes materially

## Good screenshot targets

- The new job form when explaining job settings
- The jobs list when explaining statuses and exports
- The credits page when explaining balance and purchases
- The integrations page when explaining Google Sheets or API keys

## Basic image usage

```md
![New job form](/images/example-new-job.png)
```

Use clear alt text that describes the task, not just the picture.

## When to avoid screenshots

- When the UI is changing quickly and the text alone is more stable
- When the same point is clearer as a table or short list
- When the image would mostly show placeholder or repetitive data
