## Core rule

Every page that should appear in the sidebar must be added to `docs.json`.

## Current information architecture

- `Guides` is for task-based customer workflows
- `API reference` is for endpoint-by-endpoint machine-facing docs
- Contributor pages can stay hidden, but they still need to be project-specific if they remain in the repo

## When to change navigation

- Add a page when it is a primary user task or a stable reference entry point
- Group pages by the user's goal, not by internal team ownership
- Remove or rewrite placeholder pages instead of leaving starter content hidden

## Before you merge a nav change

- Check `docs.json` for valid JSON
- Preview the sidebar locally
- Verify page ordering makes sense for a new customer
- Verify renamed or removed pages do not leave dead links behind
